# ديوني (Deyoni) - إدارة الديون والالتزامات المالية الشخصية

تطبيق آمن، مبسط، ومحلي بالكامل لإدارة وتسجيل الديون والالتزامات المعلقة (لك وعليك). تم تصميمه بهندسة برمجية واضحة ومستقلة لمساعدتك على تتبع حقوقك والتزاماتك بكل يسر وطمأنينة.

## التقنيات والخصائص المعمارية
* **لغة البرمجة:** Kotlin بالكامل بصيغة حديثة.
* **واجهة المستخدم:** Jetpack Compose لبناء واجهات غنية ومستقرة تتوافق مع معايير Material Design 3.
* **محرك قواعد البيانات:** Room Database لتأمين البيانات محلياً على جهاز العميل.
* **إصدار قاعدة البيانات:** إصدار 4، مع تفعيل تصدير المخططات (exportSchema = true) لتسهيل مراجعة هيكلية التخزين.
* **بنية التطبيق:** هندسة MVVM لتبسيط فصل منطق العمل وعرض البيانات.

## تحديثات البنية الآمنة مالياً والنسخ الاحتياطي (Stage 3)

تتضمن هذه المرحلة تحويل النسخ الاحتياطي والمزامنة إلى نظام كامل وحقيقي يعمل بكفاءة وأمان تام دون الحاجة الدائمة للاتصال بالإنترنت:

### 1. نظام محكم في الاستيراد والتصدير للنسخ الاحتياطي
* **صيغة تصدير ثرية ومكتملة:** يتم تخزين كافة التبويبات (Tabs)، والديون (Debts)، والعملاء/الأشخاص المعنيين بالدين (Persons)، والدفعات الجزئية المعالجة (Debt Payments)، وتغليفها ضمن ملف JSON موحد ومحمي.
* **تصدير بيانات الوصف التاريخي (Metadata):** يتضمن الكود الاحتياطي معلومات دقيقة وحقيقية مثل إصدار النسخة الاحتياطية (backupVersion)، وإصدار التطبيق (appVersion)، وإصدار قاعدة البيانات (databaseVersion)، ووقت وتاريخ التصدير بالملي ثانية بدقة متناهية.

### 2. واجهة استيراد متطورة بخيارين:
عند محاولة استيراد نسخة احتياطية من الحافظة، يوفر التطبيق خيارين رئيسيين للتعامل مع البيانات الموجودة حالياً:
* **الدمج الذكي (Merge Mode):** يقوم بمطابقة المعرفات والخصائص لمنع التكرار (Duplicates) وتحديث السجلات بالأحدث زمناً تلقائياً لضمان سلامة العمليات وتفادي تضارب البيانات.
* **الاستبدال الكلي (Replace Mode):** يقوم بمسح شامل وآمن لكافة الجداول الحالية داخل بيئة العمليات الذاتية (Transactions) لإحلال النسخة المستوردة بالكامل وبدقة.

### 3. بنية المزامنة السحابية الذكية ومصداقية الواجهة:
* تم استبدال عمليات ومحاكاة المزامنة المزيفة بطبقة منظمة واضحة تعكس مصداقية حالة المزامنة اللحظية (SyncState):
  * **IDLE:** وضع الاستعداد والجاهزية.
  * **SYNCING:** جاري تجهيز وفحص قنوات الاتصال بالخادم.
  * **SUCCESS:** تم المزامنة السحابية للحفظ الآمن بنجاح.
  * **FIREBASE_NOT_CONFIGURED:** تظهر هذه الحالة عندما يكون التطبيق قيد البناء محلياً بدون إعداد البنية الخلفية (ملف إعدادات Firebase). حيث توضح للمستخدم أن "المزامنة السحابية غير مفعّلة لعدم توفر ملفات تهيئة السحابية، ولكن بإمكانك الاستفادة بالكامل من نظام النسخ الاحتياطي عالي الأمان وحفظ النص المستخرج في أي مكان."
  * **FAILED:** حدوث خلل تقني أو فقدان الصلاحيات.
  * **OFFLINE:** عدم وجود إشارة اتصال حالية بالإنترنت.

إذا كنت ترغب في تشغيل مزامنة Firestore السحابية الحقيقية والمباشرة عبر حساب Google، اتبع الخطوات البسيطة التالية:

### الخطوة 1: إنشاء مشروع Firebase
1. اذهب إلى [لوحة تحكم Firebase](https://console.firebase.google.com).
2. أنشئ مشروعاً جديداً باسم `Deyoni` أو أي اسم آخر تفضله.
3. أضف تطبيق Android جديد داخل المشروع بإدخال الحزمة الرسمية للمشروع:
   `com.deoony.app`

### الخطوة 2: تهيئة ملف الإعدادات وربط التطبيق
1. قم بتحميل ملف الإعداد المولد تلقائياً **`google-services.json`**.
2. ضعه داخل مجلد التطبيق المباشر تحت المسار التالي:
   `/app/google-services.json`
3. في ملف `app/build.gradle.kts`، قم بتفعيل إضافات وجداول Firebase:
   * تفعيل ملفات الغراء: `alias(libs.plugins.google.services)` في كتلة الأدوات (plugins).

### الخطوة 3: إفساح المجال لخدمات Google Sign-In
1. انسخ المفتاح البرمجي لمعرّف العميل لـ Web Client ID المولد لك في الكونسول لخدمات Google Auth.
2. توجه في تطبيق Google AI Studio إلى نافذة الأسرار الآمنة **Secrets**، وأضف سطر بيئي باسم:
   `GOOGLE_CLIENT_ID`
   وضع قيمته هناك لضمان بقائه في مأمن، وسيتم قراءته ديناميكياً عبر `BuildConfig`.
