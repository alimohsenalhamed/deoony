# ديوني (Deyoni) - إدارة الديون والالتزامات المالية الشخصية

تطبيق آمن، مبسط، ومحلي بالكامل لإدارة وتسجيل الديون والالتزامات المعلقة (لك وعليك). تم تصميمه بهندسة برمجية واضحة ومستقلة لمساعدتك على تتبع حقوقك والتزاماتك بكل يسر وطمأنينة.

## التقنيات والخصائص المعمارية
* **لغة البرمجة:** Kotlin بالكامل بصيغة حديثة.
* **واجهة المستخدم:** Jetpack Compose لبناء واجهات غنية ومستقرة تتوافق مع معايير Material Design 3.
* **محرك قواعد البيانات:** Room Database لتأمين البيانات محلياً على جهاز العميل.
* **إصدار قاعدة البيانات:** إصدار 4، مع تفعيل تصدير المخططات (exportSchema = true) لتسهيل مراجعة هيكلية التخزين.
* **بنية التطبيق:** هندسة MVVM لتبسيط فصل منطق العمل وعرض البيانات.

## تحديثات البنية الآمنة مالياً (Stage 2)
1. **تخزين مالي آمن:** تم إلغاء استخدام قيم العشري الثنائي `Double` لتجنب أخطاء التقريب المالي الحساسة، والاستغناء عنها بتخزين ثابت بوحدة أصغر `amountMinor: Long` مع تحديد مقياس العملة `currencyScale` لضمان الدقة الحسابية القصوى.
2. **العملات المتعددة:** تفعيل حقل رمز العملة `currencyCode` لتسهيل إسناد العملة الصحيحة لكل عملية (القيمة الافتراضية للعملة المحلية: YER).
3. **تصنيف حالات الدين:** إضافة حقل حالة المعاملة `status` لدعم تحديث السجل تلقائياً بين الحالات النشطة والمدفوعة (`ACTIVE`, `PAID`, `OVERDUE`, `CANCELLED`).
4. **دقة التواريخ:** إسناد التواريخ الآمنة برقم الملي ثانية `dueDateMillis: Long?` مع الاحتفاظ بالتعبير النصي التوافقي لتواريخ السداد السابقة.
5. **الدفعات الجزئية:** بناء وتجهيز جدول وقاعدة بيانات الدفعات الجزئية المتكاملة لعمليات التوثيق المستقبلي.
6. **الترحيل الآمن للبيانات:** صياغة كود هجرة قواعد البيانات المخطط بدقة (`MIGRATION_3_4`) لتنظيم تحديث نموذج المستخدمين دون التسبب في مسح أو تلف بياناتهم المحلية.
